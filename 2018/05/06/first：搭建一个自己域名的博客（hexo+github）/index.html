first：搭建一个自己域名的博客（hexo+github）

–我的流程，仅供参考

域名问题

域名购买

　　我是在阿里云上购买的域名，搜自己喜欢，且后续成本相对划算的

控制台设置


　　12步无所谓先后，实名认证什么的先不搞不影响购买，购买的时候会提醒你创建。

　　创建购买好后，控制台各种资料，实名认证一搞。控制台左边选择域名，就可以看见你买的域名了。

GitHub操作

注册

　　在GitHub注册一个自己的账号

创建仓库

　　注意：如果你的用户名是yourname，那repository name就要填yourname.github.io。

　　这里就是刚建好的仓库

Git Bash

　　安装Git Bash，一路next安装完。

　　

NodeJs

　　安装NodeJs，因为我是用的Hexo博客框架，基于NodeJs环境的静态博客，很多npm工具可以用。
　　一路安装，在Custom Setup这步选择 add to path，省的自己再去配环境变量。　　
　　Win+r 输入cmd

　　输入node按回车，这种情况说明OK了

Hexo

文件夹

　　先新建一个文件夹，以后存放所有博客的东西。

安装

　　打开git bash，cd进入到刚新建的文件夹下

　　输入命令安装hexo。

　　这里提示一个坑点，这种情况是nodejs问题，我把nodejs卸载后重新安装了一下nodejs，然后重新安装hexo。


　　输入命令查看，出现这就安装成功了

初始化

　　输入命令进行初始化


　　初始化完成后，刚新建的文件夹里会出现很多文件。

　　这里解释一下：

  • 　　onode_modules：是依赖包
  • 　　opublic：存放的是生成的页面
  • 　　oscaffolds：命令生成文章等的模板
  • 　　osource：用命令创建的各种文章
  • 　　othemes：主题
  • 　　o_config.yml：整个博客的配置
  • 　　odb.json：source解析所得到的
  • 　　opackage.json：项目所需模块项目的配置信息
    　　然后在同样命令行中输入命令生成静态页面
